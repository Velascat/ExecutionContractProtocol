# CxRP v0.1 (frozen)

> Historical revision. The active revision is **v0.2** — see [v0.2.md](v0.2.md). This page is preserved for archival readers; the v0.1 schemas remain on disk under `cxrp/schemas/v0.1/` for backward-compatible payload validation.
>
> v0.1 was published under the project's previous name, **Execution Contract Protocol (ECP)**. The contract kinds and class names are unchanged across the rename.

CxRP v0.1 defines message contracts exchanged across OperatorConsole, OperationsCenter, and SwitchBoard.

## Scope

CxRP standardizes payload structure, shared vocabulary, and validation format.

CxRP does **not** define runtime behavior, execution engines, scheduling, lane selection algorithms, adapters, queues, routing systems, or model invocation.

## Canonical Contracts

- `TaskProposal`: requested work intent, constraints, and input references.
- `LaneDecision`: selected lane/backend and justification, produced by SwitchBoard.
- `ExecutionRequest`: bounded executable unit produced by OperationsCenter.
- `ExecutionResult`: normalized completion payload with success signal, status, and optional artifacts/diagnostics.
