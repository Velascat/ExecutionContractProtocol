# Execution Contract Protocol (ECP) v0.1

ECP v0.1 defines message contracts exchanged across OperatorConsole, OperationsCenter, and SwitchBoard.

## Scope

ECP standardizes payload structure, shared vocabulary, and validation format.

ECP does **not** define runtime behavior, execution engines, scheduling, lane selection algorithms, adapters, queues, routing systems, or model invocation.

## Canonical Contracts

- `TaskProposal`: requested work intent, constraints, and input references.
- `LaneDecision`: selected lane/backend and justification, produced by SwitchBoard.
- `ExecutionRequest`: bounded executable unit produced by OperationsCenter.
- `ExecutionResult`: normalized completion payload with success signal, status, and optional artifacts/diagnostics.
